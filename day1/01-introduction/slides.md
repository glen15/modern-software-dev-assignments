## 1. 왜 AI인가?

### 개발자의 시간 분배

```
전통적인 개발:
┌─────────────────────────────────────────┐
│ 코드 작성 (30%)  │ 디버깅 (25%)  │ 문서/회의 (45%) │
└─────────────────────────────────────────┘

AI 활용 개발:
┌─────────────────────────────────────────┐
│ 설계/검토 (40%)  │ AI 협업 (35%)  │ 문서/회의 (25%) │
└─────────────────────────────────────────┘
```

**핵심 변화**: 코드 "작성"에서 코드 "검토"로 역할 전환

---

## 2. AI 도구의 생산성 향상

### GitHub Copilot 연구 결과 (2023)

- **55% 더 빠른** 작업 완료
- **코드 품질**: 동일하거나 더 나음
- **개발자 만족도**: 88%가 생산성 향상 체감

### 실제 사용 사례

| 작업 | 전통적 방식 | AI 활용 |
|------|------------|---------|
| 보일러플레이트 코드 | 수동 작성 | 자동 생성 |
| 테스트 코드 | 시간 부족으로 생략 | 빠르게 생성 |
| 문서화 | 나중에 하자 → 안 함 | 코드와 함께 생성 |
| 버그 디버깅 | 로그 추적 | AI가 원인 분석 |

## 3. AI 도구의 종류

### 코드 자동 완성

- **GitHub Copilot**: VS Code, JetBrains 통합
- **Cursor**: AI 네이티브 에디터
- **Codeium**: 무료 대안

### AI 채팅 기반 코딩

- **Claude Code**: 터미널 기반 AI 코딩
- **ChatGPT**: 범용 AI 어시스턴트
- **Gemini**: Google의 AI 어시스턴트

### 코드 리뷰 및 분석

- **CodeRabbit**: AI 코드 리뷰
- **Semgrep**: 정적 분석 + AI

## 4. AI의 한계와 주의점

### AI가 잘하는 것

- 반복적인 코드 패턴 생성
- 문서화 및 주석 작성
- 간단한 버그 수정
- 코드 설명 및 리팩토링 제안

### AI가 못하는 것 (아직)

- 복잡한 시스템 설계
- 비즈니스 로직 이해
- 보안 취약점 완벽 탐지
- 최신 라이브러리/프레임워크 (학습 데이터 한계)

### 주의사항

```
⚠️  AI가 생성한 코드를 맹신하지 마세요
⚠️  민감한 정보를 AI에 전송하지 마세요
⚠️  저작권/라이선스 이슈를 고려하세요
⚠️  AI 결과물은 반드시 검토하세요
```

## 5. 효과적인 AI 활용 전략

### 1단계: 문제 정의

```
❌ "이거 고쳐줘"
✅ "Python에서 리스트 중복 제거하면서 순서 유지하는 함수 작성해줘"
```

### 2단계: 맥락 제공

```
❌ "로그인 기능 만들어줘"
✅ "FastAPI로 JWT 인증 기반 로그인 API 만들어줘.
    사용자 정보는 PostgreSQL에 저장되어 있고,
    비밀번호는 bcrypt로 해시되어 있어."
```

### 3단계: 점진적 개선

1. 기본 구현 요청
2. 엣지 케이스 처리 추가
3. 테스트 코드 작성
4. 리팩토링

## 6. 오늘 배울 내용 미리보기

### 프롬프팅 기초

- **K-shot 프롬프팅**: 예시를 보여주고 패턴 학습 유도
- **Chain of Thought**: 단계별 추론 유도

### 프롬프팅 심화

- **RAG**: 외부 문서를 참조하여 정확한 답변
- **Tool Calling**: AI가 외부 도구를 호출

### AI 코딩 도구

- **Cursor**: AI 네이티브 코드 에디터
- **Claude Code**: 터미널 기반 AI 에이전트

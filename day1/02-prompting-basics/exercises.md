# 프롬프팅 기초 실습

## 실습 환경 설정

### Gemini 사용 시

```bash
cd gemini
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# .env 파일에 API 키 설정
echo "GEMINI_API_KEY=your_key" > .env

# 실행
python app_gradio.py
```

### Bedrock 사용 시

```bash
cd bedrock
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# AWS CLI가 설정되어 있어야 함
aws sts get-caller-identity

# 실행
python app_gradio.py
```

## 실습 2: Chain of Thought

### 목표
단계별 사고를 유도하여 복잡한 문제를 해결하세요.

### 문제
자전거 여행 거리 계산 문제를 단계별로 풀어야 합니다.

### 단계

1. Web UI에서 **Chain of Thought** 탭 선택
2. 시스템 프롬프트 없이 실행하여 결과 확인
3. "단계별로 생각하세요" 같은 유도 문구를 시스템 프롬프트에 추가
4. 추론 과정이 명확하게 나오는지 확인

### 힌트

```
문제를 풀 때 다음 단계를 따르세요:
1. 각 날의 이동 거리를 계산하세요
2. 총 이동 거리를 구하세요
3. 최종 답을 계산하세요

단계별로 차근차근 생각해보세요.
```

### 체크리스트

- [ ] "단계별로", "차근차근" 같은 유도 문구를 사용했는가?
- [ ] 중간 계산 과정이 보이는가?
- [ ] 최종 답이 정확한가?

## 실습 3: Self-Consistency

### 목표
여러 번의 분석을 종합하여 신뢰도 높은 결론을 도출하세요.

### 문제
스타트업 아이디어의 성공 가능성을 평가합니다.

### 단계

1. Web UI에서 **Self-Consistency** 탭 선택
2. 실행 횟수를 5회로 설정
3. 시스템 프롬프트에 분석 가이드라인 추가 (힌트 참고)
4. 각 실행 결과와 종합 분석 비교

### 관찰 포인트

- 각 실행마다 어떤 점이 다른가?
- 공통적으로 언급되는 장점/단점은?
- 종합 분석이 개별 분석보다 균형 잡혀 있는가?

### 체크리스트

- [ ] 여러 번 실행 시 결과가 다양하게 나오는가?
- [ ] 종합 분석이 개별 분석을 잘 요약하는가?
- [ ] Self-Consistency의 장단점을 이해했는가?

## 자가 점검

### K-shot 프롬프팅

- [ ] Zero-shot과 Few-shot의 차이를 설명할 수 있다
- [ ] 적절한 예시 개수를 판단할 수 있다
- [ ] 예시의 다양성이 중요한 이유를 이해한다

### Chain of Thought

- [ ] CoT가 효과적인 문제 유형을 알고 있다
- [ ] 단계별 추론을 유도하는 프롬프트를 작성할 수 있다
- [ ] CoT의 한계(토큰 사용량 증가)를 이해한다

### Self-Consistency

- [ ] 종합 분석 방식의 원리를 이해한다
- [ ] 비용과 정확도의 트레이드오프를 이해한다

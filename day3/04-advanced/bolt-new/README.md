# Bolt.new 가이드: AI 기반 풀스택 앱 빌더

## 개요

Bolt.new는 자연어로 풀스택 웹 애플리케이션을 생성하는 AI 도구입니다.
프롬프트만으로 프론트엔드, 백엔드, 배포까지 한 번에 처리합니다.

---

## 학습 목표

- Bolt.new의 기능과 한계를 이해한다
- 효과적인 프롬프트로 원하는 앱을 생성할 수 있다
- 생성된 코드를 커스터마이징하고 개선할 수 있다

---

## 1. Bolt.new 소개

### Bolt.new란?

[Bolt.new](https://bolt.new)는 StackBlitz에서 만든 AI 앱 빌더입니다.

**특징**:
- 브라우저에서 바로 실행 (설치 불필요)
- 프롬프트 → 풀스택 앱 생성
- 실시간 미리보기
- 원클릭 배포

### 지원 스택

```
프론트엔드: React, Vue, Svelte, Next.js, Astro
백엔드: Node.js, API Routes
스타일링: Tailwind CSS, CSS Modules
데이터: SQLite, Supabase 연동 가능
```

## 2. 기본 사용법

### 시작하기

1. [bolt.new](https://bolt.new) 접속
2. 프롬프트 입력창에 원하는 앱 설명
3. "Create" 클릭
4. 생성 과정 확인
5. 결과 미리보기

### 첫 번째 앱 만들기

**프롬프트**:
```
할 일 관리 앱을 만들어줘.
- 할 일 추가, 완료 체크, 삭제 기능
- 로컬스토리지에 저장
- 심플한 디자인
```

**결과**:
- React + Tailwind 프로젝트 생성
- TodoList 컴포넌트
- useLocalStorage 훅
- 반응형 UI

## 3. 효과적인 프롬프트 작성

### 기본 구조

```
[앱 유형]을 만들어줘.

기능:
- [핵심 기능 1]
- [핵심 기능 2]
- [핵심 기능 3]

기술 스택:
- [프레임워크]
- [스타일링]
- [데이터 저장]

디자인:
- [스타일 키워드]
```

### 예시 1: 간단한 앱

```
날씨 대시보드를 만들어줘.

기능:
- 도시 이름으로 날씨 검색
- 현재 날씨와 5일 예보 표시
- 검색 기록 저장

기술:
- React
- OpenWeatherMap API 사용
- Tailwind CSS

디자인:
- 미니멀, 카드 기반 레이아웃
- 날씨에 따른 아이콘 표시
```

### 예시 2: 복잡한 앱

```
팀 프로젝트 관리 도구를 만들어줘.

기능:
- 프로젝트 생성/수정/삭제
- 칸반 보드 (할 일, 진행 중, 완료)
- 드래그 앤 드롭으로 상태 변경
- 팀원 할당 기능
- 마감일 설정 및 알림

기술:
- Next.js 14 (App Router)
- Tailwind CSS
- React DnD
- Zustand 상태관리
- SQLite 로컬 DB

디자인:
- Notion 스타일
- 다크모드 지원
- 반응형
```

### 예시 3: API 포함

```
간단한 블로그 플랫폼을 만들어줘.

기능:
- 글 작성 (마크다운 지원)
- 글 목록 및 상세 페이지
- 카테고리 필터링
- 검색 기능

백엔드:
- API Routes로 CRUD
- JSON 파일로 데이터 저장
- 마크다운 파싱

프론트엔드:
- Next.js
- Tailwind CSS
- React Markdown
```

## 4. 생성된 코드 활용

### 코드 다운로드

```
1. 에디터 상단의 "Export" 클릭

3. 로컬에서 압축 해제
```

### 로컬에서 실행

```bash
cd downloaded-project
npm install
npm run dev
```

### 코드 개선하기

생성된 코드를 Claude Code로 개선:

```bash
# 프로젝트 폴더에서

> 에러 핸들링을 추가해줘

> 테스트 코드를 작성해줘

> TypeScript 타입을 더 엄격하게 만들어줘
```

## 5. 고급 프롬프트 기법

### 반복적 개선

```
# 1차: 기본 생성
"할 일 앱 만들어줘"

# 2차: 기능 추가
"우선순위 기능 추가해줘 (높음/중간/낮음)"

# 3차: UI 개선
"목록을 그리드 레이아웃으로 바꾸고 애니메이션 추가해줘"

# 4차: 성능 개선
"목록이 100개 이상일 때 가상 스크롤 적용해줘"
```

### 구체적인 컴포넌트 요청

```
React 데이터 테이블 컴포넌트를 만들어줘.

Props:
- data: 배열 데이터
- columns: 컬럼 정의
- sortable: 정렬 가능 여부
- searchable: 검색 가능 여부

기능:
- 컬럼 헤더 클릭으로 정렬
- 전체 텍스트 검색
- 페이지네이션 (10, 25, 50개)
- 행 선택 (체크박스)
- CSV 내보내기

TypeScript로 타입 정의 포함해줘.
```

### 기존 코드와 통합

```
기존 React 프로젝트에 추가할 알림 시스템을 만들어줘.

요구사항:
- Toast 형태의 알림
- 성공/경고/에러/정보 4가지 타입
- 자동 사라짐 (5초)
- 수동 닫기 버튼
- 애니메이션 (슬라이드 인/아웃)

Context API로 전역 상태 관리.
useNotification 훅 제공.
```

## 6. 한계와 대응 방법

### Bolt.new의 한계

| 한계 | 대응 방법 |
|------|----------|
| 복잡한 백엔드 로직 | 기본 생성 후 수동 확장 |
| 데이터베이스 연동 | Supabase 등 BaaS 활용 |
| 인증/권한 | Auth0, Clerk 등 연동 |
| 테스트 코드 미포함 | 수동 추가 또는 AI로 생성 |
| 대규모 프로젝트 | 모듈별로 생성 후 통합 |

### 대응 예시: 인증 추가

```
# Bolt.new로 기본 앱 생성 후

# 1. Clerk 연동 요청
"이 앱에 Clerk 인증 추가해줘.
- 로그인/회원가입 페이지
- 보호된 라우트
- 사용자 프로필"

# 2. 또는 수동으로 추가
npm install @clerk/nextjs
# Claude Code로 연동 코드 생성
claude
> Clerk을 이 Next.js 프로젝트에 연동해줘
```

## 7. 실전 워크플로우

### 빠른 프로토타입 → 프로덕션

```
1단계: Bolt.new로 프로토타입
  └── 핵심 기능만 포함한 MVP 생성

2단계: 코드 다운로드 및 검토
  └── 생성된 구조와 패턴 이해

3단계: Claude Code로 개선
  └── 타입 강화, 에러 처리, 테스트

4단계: 추가 기능 구현
  └── 인증, DB 연동, 최적화

5단계: 배포
  └── Vercel, Netlify 등 활용
```

### 예시: 대시보드 앱

```
# 1단계: Bolt.new
"관리자 대시보드를 만들어줘.
- 사이드바 네비게이션
- 통계 카드
- 차트 (라인, 바)
- 데이터 테이블
더미 데이터 사용해줘."

# 2단계: 다운로드 후
unzip dashboard-project.zip
cd dashboard-project
npm install
code .  # VSCode에서 열기

# 3단계: Claude Code로 개선
claude
> package.json을 보고 이 프로젝트의 기술 스택 분석해줘
> Chart 컴포넌트에 로딩 상태와 에러 처리 추가해줘
> API 연동을 위한 서비스 레이어 만들어줘

# 4단계: 실제 데이터 연동
claude
> 더미 데이터를 실제 API 호출로 바꿔줘.
  API 엔드포인트는 /api/stats, /api/users
```

## 8. Bolt.new vs 기타 도구 비교

| 기능 | Bolt.new | v0.dev | Claude Code |
|------|----------|--------|-------------|
| **접근 방식** | 풀스택 앱 생성 | UI 컴포넌트 생성 | 코드 수정/생성 |
| **사용 환경** | 브라우저 | 브라우저 | 터미널 |
| **출력** | 전체 프로젝트 | 단일 컴포넌트 | 파일 수정 |
| **커스터마이징** | 프롬프트로 | 프롬프트로 | 직접 제어 |
| **적합한 용도** | 프로토타입 | UI 스니펫 | 기존 프로젝트 |

### 조합 활용

```
1. v0.dev로 개별 UI 컴포넌트 생성
2. Bolt.new로 전체 앱 스캐폴딩
3. Claude Code로 통합 및 개선
```

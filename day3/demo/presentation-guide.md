# 발표 가이드

## 개요

- **발표 시간**: 팀당 5분
- **형식**: 라이브 데모 + 설명
- **발표자**: 팀장 또는 지정된 팀원

---

## 발표 구성 (5분)

### 1. 소개 (30초)

- 팀명
- 프로젝트명
- 한 줄 소개

```
예시:
"안녕하세요, 저희는 팀 '코드브레이커'입니다.
저희가 만든 '프롬프트 저장소'는 AI 프롬프트를 저장하고 관리하는 앱입니다."
```

### 2. 문제 정의 (30초)

- 어떤 문제를 해결하나요?
- 왜 이 프로젝트를 선택했나요?

```
예시:
"AI 도구를 사용하다 보면 좋은 프롬프트를 잊어버리거나
다시 찾기 어려운 경우가 많습니다.
저희는 이 문제를 해결하고자 했습니다."
```

### 3. 라이브 데모 (2분 30초)

핵심 기능 시연:

1. 주요 기능 1 시연
2. 주요 기능 2 시연
3. 특별한 기능 (있다면)

**데모 팁**:
- 미리 테스트 데이터 준비
- 인터넷 문제 대비 로컬 실행
- 느린 동작 대비 말로 설명 준비

### 4. 기술 스택 & 배운 점 (1분)

- 사용한 기술
- Day 1-2 학습 내용 적용
- 어려웠던 점과 해결 방법

```
예시:
"백엔드는 FastAPI, 프론트는 Vanilla JS를 사용했습니다.
Day 2에서 배운 Semgrep으로 XSS 취약점을 발견하고 수정했습니다.
처음에 CORS 설정에서 막혔는데, AI 도구의 도움으로 해결했습니다."
```

### 5. 마무리 (30초)

- 향후 계획 (확장 가능성)
- 감사 인사

# 프롬프팅 치트시트

## 기본 원칙

### 좋은 프롬프트의 특징

1. **구체적**: 원하는 것을 명확하게
2. **맥락 제공**: 필요한 배경 정보 포함
3. **형식 지정**: 출력 형식 명시
4. **예시 제공**: K-shot으로 패턴 제시

---

## 프롬프팅 기법

### 1. Zero-shot

예시 없이 직접 요청

```
텍스트의 감정을 분석해줘: "오늘 정말 기분이 좋다"
```

### 2. Few-shot (K-shot)

예시를 제공하여 패턴 학습

```
다음 형식으로 감정을 분석해줘:

텍스트: "기쁘다" → 감정: 긍정
텍스트: "슬프다" → 감정: 부정
텍스트: "오늘 정말 기분이 좋다" → 감정: ?
```

### 3. Chain of Thought (CoT)

단계별 추론 유도

```
단계별로 생각해서 풀어줘:

문제: 3^10 mod 7 = ?

1단계: 3의 거듭제곱 패턴 찾기
2단계: ...
```

### 4. Self-Consistency

여러 번 추론 후 다수결

```
이 문제를 5번 풀고 가장 많이 나온 답을 선택해줘.
```

```

```

```

### JSON 형식

```

  "summary": "요약",
  "key_points": ["포인트1", "포인트2"],
  "sentiment": "긍정/부정/중립"

```

```
```

## 고급 기법

```

문서에 없는 내용은 "문서에 없습니다"라고 해줘.

문서:
[문서 내용]

질문: [질문]
```

### Tool Calling

```
다음 도구를 사용할 수 있어:
- search(query): 웹 검색
- calculate(expression): 수식 계산

도구를 사용하려면 JSON으로 응답해:
{"tool": "도구명", "args": {"인자": "값"}}
